# Below is a professional `SECURITY.md` suitable for a public portfolio repository.

```markdown
# Security Policy

## Security Commitment

The security of this project is a priority. Although this repository primarily hosts my personal portfolio website, I appreciate the responsible disclosure of any security vulnerabilities or potential security concerns.

If you discover a vulnerability, please report it privately rather than creating a public issue.

---

## Supported Versions

The latest version of this repository is actively maintained and supported.

| Version | Supported |
| -------- | :-------: |
| Latest (main) | ✅ |
| Previous versions | ❌ |

---

## Reporting a Vulnerability

If you believe you have discovered a security vulnerability, please report it responsibly by contacting me directly.

**Email:** dhruv.dhayal@s.amity.edu

Please include the following information whenever possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Screenshots or proof of concept (if applicable)
- Suggested mitigation (optional)

Please avoid publicly disclosing vulnerabilities until they have been reviewed and addressed.

---

## Response Timeline

Every report is handled responsibly and reviewed as quickly as possible.

| Stage | Expected Time |
|--------|---------------|
| Initial acknowledgement | Within 48 hours |
| Investigation | 3–7 business days |
| Resolution (if applicable) | Depends on severity and complexity |

These timeframes are best-effort estimates and may vary depending on the nature of the report.

---

## Scope

This policy applies to:

- Source code
- Website functionality
- Client-side scripts
- Configuration files
- Third-party integrations included in this repository

Out of scope:

- Vulnerabilities in third-party services or libraries that are unrelated to this repository
- Social engineering attacks
- Physical attacks
- Denial-of-Service (DoS/DDoS) testing
- Automated vulnerability scanning that negatively impacts service availability

---

## Responsible Disclosure

Please:

- Provide sufficient detail for reproduction.
- Allow reasonable time for investigation and remediation.
- Avoid publicly sharing details before a fix is available.
- Act in good faith and avoid accessing or modifying data that does not belong to you.

---

## Security Best Practices

This project follows modern development practices, including:

- HTTPS deployment
- Dependency management
- Secure Git workflows
- Environment variable separation for sensitive configuration
- Regular dependency updates
- Version control with Git
- Code reviews before major releases

---

## Acknowledgements

Responsible security disclosures are greatly appreciated. Individuals who report legitimate security issues may be acknowledged in future project releases, unless they prefer to remain anonymous.

---

Thank you for helping improve the security and reliability of this project.
```
